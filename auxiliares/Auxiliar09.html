Descriptores

Nos permiten mantener representaciones de determinadas caractersticas de los objetos. Es posible que queramos representar el objeto completo, o bien localmente sus propiedades.

Un descriptor sencillo es utilizar los eigenvalues del laplaciano

Sabemos que el operador de Laplace-Beltrami es invariante a las isometras, por lo que describir un objeto utilizando sus propiedades resultara en descriptores similares para objetos similares. Para esto, podemos utilizar por ejemplo los eigenvalues.

shapeDNA.py:

  • Calculamos la matriz laplaciana
  • Calculamos eigenvalues a partir de ella

Si bien para cada objeto tenemos $n$ valores propios, podemos visualizarlos todos en un mismo grfico con una curva. As, podemos comparar los descriptores para distintos objetos.

Heat Kernel Signature

Tambin es posible ver cmo se distribuye el calor en el Manifold, utilizando el laplaciano. Esta es una forma de describir los objetos donde tendremos informacin de, para cada vrtice y dado un tiempo t, su valor. Podemos imaginarlo como cuanto calor permanece en cada vrtice luego de un tiempo $t$.

Veamos feature_point_hks.py.

Primero, computamos el HKS para un solo tiempo. Esto significa que nuestro descriptor tendr un valor de informacin para cada uno de los vrtices. Podemos utilizar esta informacin para extraer puntos de inters, por ejemplo aquellos puntos donde el proceso de difusin de calor acumul ms cantidad.

Podemos determinar fcilmente estos vrtices, pues basta comparar el descriptor de cada vrtice con el de sus vecinos.

Pueden probar ir variando en qu t calculamos las caractersticas. La siguiente imagen ilustra el caso con $t=0.001$.

Por supuesto, podemos describir un periodo de tiempo amplio con este descriptor, no es necesario quedarse con un solo $t$. El incluir muchos valores de $t$ nos permite caracterizar de manera ms completa el proceso de difusin de calor.

En heat_kernel_signature.py, calculamos 100 valores de $t$, y graficamos estos 100 valores para 4 vrtices del objeto.

Para visualizar el descriptor para todos los vrtices, podemos aadir una cantidad escalar por cada dimensin del hks. Esto lo podemos hacer a partir del cdigo de heat_kernel_signature.py.

Segmentacin con Global Point Signature

(Global Point Signature): $$ \left( \frac{\phi(\vec{p_1})}{\lambda_1}, \frac{\phi(\vec{p_2})}{\lambda_2}, ..., \frac{\phi(\vec{p_m})}{\lambda_m} \right) $$

Algo que se puede hacer con el descriptor GPS es utilizar las descripciones de los vrtices para segmentar los objetos. Podemos considerar las descripciones para cada vrtice como elementos m-dimensionales (ya que este tipo de descriptores utilizan $m$ valores y vectores propios) y agruparlos viendo qu vertices contienen propiedades similares. Un simple proceso de clustering basta para esto, como podemos experimentar en gps.py.
